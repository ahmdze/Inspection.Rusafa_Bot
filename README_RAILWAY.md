# دليل رفع البوت على Railway مع إعداد قاعدة البيانات

## 📋 المتطلبات المسبقة

1. **حساب GitHub** - لرفع الكود
2. **حساب Railway** - للاستضافة (https://railway.app)
3. **Telegram Bot Token** - من @BotFather
4. **Admin IDs** - أرقام هوية المديرين على Telegram

---

## 🚀 الخطوة 1: تحضير الكود

### 1.1 تأكد من الملفات الأساسية

يجب أن يحتوي مشروعك على الملفات التالية:

```
/workspace/
├── bot.py                 # ملف البوت الرئيسي
├── database.py            # وحدة قاعدة البيانات (محدثة لدعم PostgreSQL)
├── requirements.txt       # المكتبات المطلوبة
├── Procfile              # ملف تعريف العملية لـ Railway
├── runtime.txt           # إصدار Python
└── .env                  # متغيرات البيئة (للتطوير المحلي فقط)
```

### 1.2 تحديث `.gitignore`

تأكد من تجاهل ملف `.env` وقاعدة بيانات SQLite:

```gitignore
.env
*.sqlite
*.db
__pycache__/
*.pyc
```

---

## 🗄️ الخطوة 2: إعداد قاعدة البيانات على Railway

### 2.1 إنشاء قاعدة بيانات PostgreSQL جديدة

1. اذهب إلى [Railway Dashboard](https://railway.app/dashboard)
2. اضغط على **"New Project"**
3. اختر **"Deploy from GitHub repo"** أو **"Start a new project"**
4. بعد إنشاء المشروع، اضغط على **"New"** → **"Database"** → **"PostgreSQL"**

### 2.2 الحصول على رابط قاعدة البيانات

1. بعد إنشاء قاعدة البيانات، اضغط عليها
2. اذهب إلى تبويب **"Variables"**
3. انسخ قيمة `DATABASE_URL` (ستكون بهذا الشكل):
   ```
   postgresql://postgres:[password]@[host]:[port]/[database_name]
   ```

---

## 🔧 الخطوة 3: إعداد متغيرات البيئة على Railway

في نفس تبويب **"Variables"** لمشروعك على Railway، أضف المتغيرات التالية:

| المتغير | القيمة | ملاحظات |
|---------|--------|---------|
| `TOKEN` | `your_bot_token_here` | التوكن من @BotFather |
| `ADMIN_IDS` | `5372786095,123456789` | أرقام المديرين مفصولة بفواصل |
| `DATABASE_URL` | `postgresql://...` | الرابط من الخطوة 2.2 |
| `LOG_LEVEL` | `INFO` | مستوى السجلات (اختياري) |

⚠️ **مهم**: لا ترفع ملف `.env` إلى GitHub! استخدم متغيرات Railway فقط.

---

## 📤 الخطوة 4: رفع الكود إلى GitHub

### 4.1 تهيئة Git (إذا لم تكن مهيأة)

```bash
cd /workspace
git init
git add .
git commit -m "Initial commit with PostgreSQL support"
```

### 4.2 إنشاء مستودع جديد على GitHub

1. اذهب إلى https://github.com/new
2. أنشئ مستودعاً جديداً (سمّه مثلاً `inspection-bot`)
3. انسخ رابط المستودع

### 4.3 ربط المستودع ورفعه

```bash
git remote add origin https://github.com/YOUR_USERNAME/inspection-bot.git
git branch -M main
git push -u origin main
```

---

## 🎯 الخطوة 5: ربط Railway بـ GitHub

### 5.1 الربط الأولي

1. في Railway Dashboard، اضغط على **"New"** → **"GitHub Repo"**
2. اختر المستودع الذي رفعته (`inspection-bot`)
3. سيبدأ Railway تلقائياً في بناء وتشغيل البوت

### 5.2 إضافة متغيرات البيئة

1. بعد بدء البناء، اضغط على مشروعك
2. اذهب إلى تبويب **"Variables"**
3. أضف جميع المتغيرات من الخطوة 3

---

## ✅ الخطوة 6: التحقق من التشغيل

### 6.1 مراقبة السجلات

1. في مشروع Railway، اذهب إلى تبويب **"Deployments"**
2. اضغط على الـ Deployment النشط
3. راقب السجلات للتأكد من:
   - ✅ اتصال قاعدة البيانات ناجح
   - ✅ البوت بدأ العمل
   - ✅ لا توجد أخطاء

### 6.2 اختبار البوت

1. افتح Telegram وابحث عن البوت الخاص بك
2. أرسل `/start`
3. يجب أن يعمل البوت بشكل طبيعي

---

## 🔍 استكشاف الأخطاء

### مشكلة: خطأ في الاتصال بقاعدة البيانات

**الحل:**
- تأكد من صحة `DATABASE_URL`
- تحقق من أن PostgreSQL تم إنشاؤها بنجاح
- تأكد من وجود `psycopg2-binary` في `requirements.txt`

### مشكلة: البوت لا يستجيب

**الحل:**
- تحقق من صحة `TOKEN`
- راجع السجلات في Railway للأخطاء
- تأكد من أن `ADMIN_IDS` صحيحة

### مشكلة: خطأ في الجداول

**الحل:**
- البوت سينشئ الجداول تلقائياً عند أول تشغيل
- إذا استمرت المشكلة، احذف قاعدة البيانات وأنشئها مجدداً
