# توثيق بوت التفتيش المدرسي

## نظرة عامة

بوت Telegram متخصص في إدارة زيارات التفتيش المدرسي، يسمح للمفتشين بتسجيل الملاحظات وإنشاء تقارير احترافية.

## الميزات الرئيسية

### 1. إدارة الزيارات
- إنشاء زيارات ميدانية جديدة
- تسجيل ملاحظات تفصيلية حسب المحاور
- إرفاق صور ومستندات (حتى 500 صورة دفعة واحدة)
- حفظ المسودات تلقائياً

### 2. التقارير
- توليد تقارير Word احترافية بتنسيق RTL
- إحصائيات شاملة عن الزيارات
- تصدير البيانات بصيغ متعددة

### 3. الصلاحيات
- **مدير (Admin)**: وصول كامل، إغلاق الزيارات، عرض جميع التقارير
- **عضو (Member)**: إنشاء زيارات، إضافة ملاحظات، عرض تقاريره فقط

### 4. الخصوصية والأمان
- حذف بيانات المستخدم عند الطلب
- تسجيل جميع العمليات (Audit Log)
- تشفير معرفات المسؤولين
- تنظيف تلقائي للبيانات القديمة

## التثبيت والإعداد

### 1. تثبيت المتطلبات

```bash
pip install python-telegram-bot python-docx Pillow requests
```

### 2. إعداد المتغيرات البيئية

أنشئ ملف `.env` في المجلد الرئيسي:

```env
TELEGRAM_BOT_TOKEN=your_bot_token_here
ADMIN_IDS=123456789,987654321
DATABASE_URL=inspection_bot.db
LOG_LEVEL=INFO
```

### 3. تشغيل البوت

```bash
python bot.py
```

## قاعدة البيانات

### الجداول الرئيسية

#### Users
- `id`: المعرف الفريد
- `telegram_id`: معرف Telegram
- `username`: اسم المستخدم
- `full_name`: الاسم الكامل
- `role`: الدور (admin/member)
- `created_at`: تاريخ الإنشاء
- `consent_given`: الموافقة على الخصوصية

#### Visits
- `id`: المعرف الفريد
- `school_name`: اسم المدرسة
- `visit_date`: تاريخ الزيارة
- `created_by`: منشئ الزيارة
- `status`: الحالة (open/closed)
- `created_at`: تاريخ الإنشاء

#### Observations
- `id`: المعرف الفريد
- `visit_id`: معرف الزيارة
- `axis`: المحور
- `description`: الوصف
- `status`: الحالة
- `created_at`: تاريخ الإنشاء

#### Attachments
- `id`: المعرف الفريد
- `observation_id`: معرف الملاحظة
- `file_id`: معرف الملف في Telegram
- `file_type`: نوع الملف
- `file_path`: مسار الملف
- `created_at`: تاريخ الإنشاء

#### Audit_Log
- `id`: المعرف الفريد
- `user_id`: معرف المستخدم
- `action`: الإجراء
- `details`: التفاصيل
- `timestamp`: الوقت

### الفهارس (Indexes)

تم إضافة فهارس لتحسين الأداء على:
- `Users.telegram_id`
- `Visits.created_by`
- `Visits.status`
- `Observations.visit_id`
- `Attachments.observation_id`
- `Audit_Log.user_id`
- `Audit_Log.timestamp`

## التخزين المؤقت (Cache)

يستخدم البوت نظام تخزين مؤقت لتحسين الأداء:

```python
from cache_manager import get_cache

cache = get_cache(ttl=300)  # 5 دقائق

# تخزين قيمة
cache.set("key", "value")

# استرجاع قيمة
value = cache.get("key")

# حذف قيمة
cache.delete("key")

# إحصائيات
stats = cache.get_stats()
```

### الإحصائيات المتاحة
- `size`: عدد العناصر الحالية
- `hits`: عدد مرات الاسترجاع الناجح
- `misses`: عدد مرات عدم العثور
- `hit_rate`: نسبة النجاح (%)
- `expired_count`: عدد العناصر منتهية الصلاحية

## الأمان

### أفضل الممارسات المطبقة

1. **حماية التوكن**
   - تخزين في ملف `.env`
   - عدم تضمينه في الكود
   - صلاحيات قراءة محدودة للملف

2. **حماية ADMIN_IDS**
   - تجزئة قبل التخزين
   - التحقق عند كل عملية

3. **SQL Injection**
   - استخدام معاملات (?) في جميع الاستعلامات
   - لا توجد استعلامات ديناميكية غير آمنة

4. **Audit Logging**
   - تسجيل جميع العمليات الحساسة
   - تتبع تغييرات الصلاحيات
   - مراقبة حذف البيانات

### الخصوصية والامتثال

- **موافقة المستخدم**: تسجيل `consent_given` عند أول استخدام
- **حذف البيانات**: دالة `delete_user_data()` لحذف جميع بيانات المستخدم
- **الشفافية**:用户可以请求查看其存储的数据

## معالجة الأخطاء

### مستويات التسجيل

```python
import logging

logger = logging.getLogger('inspection_bot')

logger.debug("رسالة تصحيح")      # تفاصيل دقيقة
logger.info("رسالة معلومات")     # أحداث عادية
logger.warning("رسالة تحذير")    # مشاكل محتملة
logger.error("رسالة خطأ")        # أخطاء فعلية
logger.critical("رسالة حرجة")    # أخطاء كارثية
```

### إعادة المحاولة (Retry)

يتم تطبيق آلية إعادة المحاولة لطلبات Telegram API:

```python
max_retries = 3
for attempt in range(max_retries):
    try:
        # محاولة إرسال الرسالة
        await context.bot.send_message(...)
        break
    except TelegramError as e:
        if attempt == max_retries - 1:
            logger.error(f"فشل بعد {max_retries} محاولات: {e}")
            raise
        time.sleep(2 ** attempt)  # انتظار متزايد
```

## الانتقال إلى PostgreSQL

للإنتاج عالي الحمل، يُوصى بالانتقال إلى PostgreSQL:

### 1. تثبيت المكتبة

```bash
pip install psycopg2-binary
```

### 2. تحديث DATABASE_URL

```env
DATABASE_URL=postgresql://user:password@localhost:5432/inspection_bot
```

### 3. تعديلات قاعدة البيانات

```sql
-- تمكين الاتصالات المتوازية
ALTER SYSTEM SET max_connections = 200;

-- إضافة Connection Pooling
-- (يتم التعامل معه في الكود عبر sqlalchemy أو psycopg2.pool)
```

### 4. فوائد PostgreSQL

- دعم الكتابة المتوازية
- أداء أفضل مع البيانات الكبيرة
- ميزات متقدمة (Full-text search, JSONB)
- نسخ احتياطي أسهل
